## File Naming Convention
Pattern: `day-talk_number-author-slug_of_the_talk.md`

### Examples:
- Day 1, Talk 4: `1-04-aden-buie-tools-llms-humans-r.md`
- Day 2, Talk 1: `2-01-maxwell-geodl-geospatial.md`

## Quick Create Commands
When ready for next talk, use pattern:
- Extract key words from title for slug
- Use speaker's last name (first name if multiple authors)
- Keep slug concise but descriptive

## Template Structure
Each file should follow the established pattern:
1. **Talk Information** - Speaker, time, context
2. **Abstract/Summary** - Official description
3. **Conference Context** - Position in program
4. **Key Points & Analysis** - Live notes during talk
5. **Technical Content** - Code, methods, tools
6. **Research Implications** - Broader significance
7. **Q&A Session** - Questions and responses
8. **Questions for Further Investigation** - Follow-up research
9. **Resources & References** - Links, tools, packages
10. **Action Items** - Post-conference follow-up
11. **Notes** - Status and context
